# Contributing to MeetScribe AI

We love your input! We want to make contributing to MeetScribe AI as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## Development Process

We use GitHub to host code, to track issues and feature requests, as well as accept pull requests.

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!

## Pull Request Guidelines

- **Atomic Commits**: Keep your commits small and focused on a single change.
- **Descriptive Titles**: Use clear, concise titles for your pull requests.
- **Documentation**: Update relevant documentation (README, API docs) for any significant changes.

## Code of Conduct

Help us keep MeetScribe AI open and inclusive. Please read and follow our [Code of Conduct](CODE_OF_CONDUCT.md).
